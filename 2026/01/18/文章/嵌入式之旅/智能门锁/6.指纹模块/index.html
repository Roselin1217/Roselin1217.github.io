6.指纹模块

RossLin2026-01-18

AS608 指纹识别模块

1. 硬件接口与初期测试

在编写驱动代码前，需确保模块硬件正常且通讯链路通畅。

WAK 引脚（感应输出）

  • 作用：用于判断传感器上是否有手指按下。
  • 状态：高电平表示有手指触摸，低电平表示空闲。

模块测试思路

  • 首选上位机测试：使用 USB 转 TTL 模块连接电脑。
  • 上位机验证：使用厂家提供的测试软件确认模块能正常录入和搜索指纹。
  • 波特率匹配：确保串口与指纹模块波特率一致。若不一致会导致通讯乱码或无法解析。
  • 调整建议：可通过上位机预先调整好模块的串口波特率。

2. 串口通讯进阶调试

在单片机开发中，串口调试的准确性直接影响协议解析。

printf 的局限性（二进制通讯陷阱）

  • 问题描述：使用 printf(“%s”, string) 打印数据时，若数据包中间包含字节 0x00，打印会直接截断。
  • 原因分析：C 语言中 %s 将 0x00 识别为字符串结束符（Null Terminator）。
  • 协议特性：AS608 基于二进制数据包，0x00 经常出现在包长度或校验和的高位，因此严禁使用字符串处理函数调试。

封装发送函数

建议通过循环指定长度来确保每个字节（包括 0x00）都能被正确发出。

3. AS608 核心业务逻辑实现

指纹模块的操作本质上是指令包的有序交互。

常用指令集参考

函数名 指令代码 功能描述
PS_GetImage 01H 从传感器读入图像存入 ImageBuffer
PS_GenChar 02H 根据图像生成指纹特征存入 CharBuffer
PS_RegModel 05H 合并两个特征缓冲区生成模板
PS_StoreChar 06H 将模板储存到 Flash 指纹库
PS_Search 04H 搜索指纹库中的匹配项

指纹处理流程模拟

通常在 as608_proc 处理函数中分为注册和识别两种模式。

  • 注册模式（两次采样确认）：

    1. 第一次采集：PS_GetImage -> PS_GenChar(1)

    2. 第二次采集：PS_GetImage -> PS_GenChar(2)

    3. 合并与存储：PS_RegModel（合成模板）-> PS_StoreChar(addr)（存入指定 ID 槽位）

  • 验证模式（搜索）：

    1. 采集特征：PS_GetImage -> PS_GenChar(2)
    2. 搜索比对：执行 PS_Search
    3. 分数判断：解析应答包中的得分字节（如 uart7_rec_string[13]）
    4. 逻辑处理：得分 > 50 视为匹配成功（执行开锁）；得分 < 50 视为失败（播放提示音）

4. 调试要点总结

  • 确认码检查：每发送一条指令，必须检查返回包中的确认码。只有确认码为 00H 才表示当前步骤执行成功。
  • WAK 状态检测：在流程开始前，先读取 WAK 引脚电平。仅在有手指触摸时才启动指令流，避免单片机无效循环串口指令。
