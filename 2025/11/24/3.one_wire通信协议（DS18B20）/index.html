单总线协议详解（DS18B20 ）

一、单总线协议基础

1.1 什么是单总线？

单总线（1-Wire） 是 Dallas/Maxim 公司开发的一种串行通信协议。

核心特点：

  • ✅ 只需 1 根数据线 Data Queue （DQ）+ 1 根地线（GND）
  • ✅ 支持多设备挂载（每个设备有唯一 64 位 ROM 码）
  • ✅ 半双工通信（同一时刻只能单向传输）
  • ✅ 主从架构（MCU 是主机，传感器是从机）

1.2 开漏输出原理（重要！）

单总线使用 开漏输出（Open-Drain） 模式：

MCU 操作 DQ 引脚状态 实际电平
DQ = 0 强制拉低 低电平（0V）
DQ = 1 释放总线（高阻态） 高电平（由上拉电阻决定）

关键理解：

  • DQ = 1 不是主动输出高电平，而是”松手”让上拉电阻把总线拉高
  • 从机可以在总线被释放后拉低总线（实现数据传输）

3.2 ROM 操作指令（所有单总线设备通用）

指令码 名称 功能 应用场景
0xCC Skip ROM 跳过 ROM 匹配 单设备系统（最常用）
0x55 Match ROM 选择特定设备 多设备总线
0x33 Read ROM 读取 64 位 ROM 码 获取设备 ID
0xF0 Search ROM 搜索所有设备 设备枚举

四、DS18B20 专属内容

4.1 功能指令（DS18B20 特定）

指令码 名称 功能
0x44 Convert T 启动温度转换（耗时 750ms）
0xBE Read Scratchpad 读取暂存器（含温度数据）
0x4E Write Scratchpad 写入配置（精度、报警阈值）

4.2 暂存器结构（9 字节）

调用 0xBE 后，DS18B20 按顺序发送：

字节索引 内容 说明
Byte 0 温度低 8 位（LSB） 第1次 Read_DS18B20()
Byte 1 温度高 8 位（MSB） 第2次 Read_DS18B20()
Byte 2 TH 寄存器 高温报警阈值
Byte 3 TL 寄存器 低温报警阈值
Byte 4 配置寄存器 精度设置（9/10/11/12位）
Byte 5-7 保留 固定 0xFF
Byte 8 CRC 校验 数据完整性校验
