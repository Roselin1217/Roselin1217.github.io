8.任务调度器

RossLin2025-11-24

C语言结构体与任务调度器

第一部分：C语言结构体基础

结构体是C语言中用于将不同类型的数据组合在一起的复合数据类型。在嵌入式开发中，它是实现多任务调度、管理复杂数据的基础。

2. 声明变量的三种方式

方式一：先定义类型，再声明变量（标准写法）

方式二：使用 typedef 定义别名（推荐写法）

这样可以省略 struct 关键字，代码更简洁。

方式三：指定成员初始化（C99标准）

优点是初始化顺序不影响代码逻辑，可读性强。

3. 成员访问规则

  • . 点运算符：用于结构体变量。
  • -> 箭头运算符：用于结构体指针。

第二部分：任务调度器实战

1. 定义结构体（画一张“任务卡”的模板）

这是调度器的核心数据结构，每个任务卡上记录了三个关键信息：

  1. 要做什么（函数指针）
  2. 多久做一次（执行周期）
  3. 上次什么时候做的（时间戳）

2. 创建任务列表（填写8张任务卡）

利用结构体数组来存储所有任务。推荐使用指定成员初始化（.成员名 = 值），这样即使以后结构体成员顺序变了，代码也不会出错。

3. 初始化：自动计算任务数量

不要手动去数数组里有几个任务，让计算机帮你算。这样以后增删任务时，不需要修改这里的代码。

4. 核心运行函数（调度引擎）

这个函数通常放在主循环 while(1) 中。它会不断扫描任务列表，判断时间是否到了。

5. 完整使用流程

第三部分：重点速查表

1. 重点口诀（避坑指南）

项目 正确写法 错误写法 原因解析
取成员 Scheduler_Task[i].task_func() Scheduler_Task[i]->task_func() Scheduler_Task[i] 是数组里的实体，必须用点号 .
初始化 { .task_func = xxx, .rate_ms = yyy } 乱序写且不带点号 指定成员初始化最安全，防止结构体成员顺序调整后程序崩溃
调用函数 task_func() task_func 不加括号只是获取函数地址，加括号才是执行函数
计算数量 sizeof(arr)/sizeof(task_t) 手动写数字 8 自动计算，以后增删任务不需要改代码

2. 数据类型速查表

类型 占用字节 范围（无符号） 典型用途 代码示例
unsigned char 1 0 ~ 255 普通标志位、端口、循环变量 unsigned char i;
unsigned int 2 0 ~ 65,535 短延时、ADC数据采集 unsigned int ad_val;
unsigned long int 4 0 ~ 4,294,967,295 时间戳、长计时 uwTick, last_ms
unsigned long 4 同上 省略写法，效果一样 unsigned long time;
