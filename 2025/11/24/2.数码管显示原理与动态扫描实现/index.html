**数码管和定时器 **

一、数码管基本原理

1. 数码管结构

数码管本质上是由8个LED段（ag + 小数点dp）组合而成的显示器件，用于显示数字09及部分字母。

根据内部连接方式，可分为两种类型：

类型 阳极连接 阴极连接 控制方式 亮灭逻辑
共阳极数码管 所有LED阳极连在一起接VCC 每段独立接驱动电路 控制阴极电平 0亮，1灭
共阴极数码管 每段独立接驱动电路 所有LED阴极连在一起接地 控制阳极电平 1亮，0灭

✅ 本系统使用的是共阳极数码管（根据电路图与代码中0xc0, 0xf9...推断）
因此：输出低电平 → 段点亮；输出高电平 → 段熄灭

3. 什么是“消影”？如何解决？

❓ 什么是消影？

在动态扫描显示中，当从一个数码管切换到另一个时，由于段码数据提前加载或位选信号延迟关闭，会导致前一个数码管在切换瞬间仍有残影（部分段微亮），这种现象称为**“消影”**（或“鬼影”）。

✅ 消影产生的原因：

  • 位选（选中哪个数码管）与段选（显示什么数字）未严格同步；
  • 段码数据写入后，未先关闭所有位选就切换，造成短暂重叠。

🔧 解决方法：

“先关位选 → 再送段码 → 再开位选”
即：消影三步法

  1. 先关闭所有数码管（P0 = 0xFF，段码全灭）
  2. 再进行位选（选中要显示的数码管）
  3. 最后写入段码并锁存

✅ 本系统通过在每次显示前先执行一次**“全灭”操作**，有效消除鬼影。

二、数码管动态显示原理

🎯 原理概述

动态显示是指多个数码管轮流点亮，利用人眼的视觉暂留效应（约10~40ms），在快速切换下产生“同时点亮”的错觉。

  • 刷新频率建议：> 50Hz（即每50ms刷新一次），人眼无闪烁。

  • 每个数码管点亮时间：约 1~5ms

  • 优点：节省IO口、功耗低、电路简单

  • 缺点：需程序定时刷新，占用CPU资源

✅ 动态扫描实现步骤

“消影 → 位选 → 段选”

步骤 操作 目的
1. 消影 P0 = 0xFF + 锁存 关闭所有段，避免残影
2. 位选 选中某个数码管（如第3位） 让电流只流过该数码管
3. 段选 写入对应数字的段码 显示所需数字

💡 注意：位选与段选需分别通过不同的锁存器控制（本系统使用两个锁存器：一个控制位选、一个控制段选）

三、硬件连接说明

  • P0口：输出段码与位选信号（共用）
  • P2口:控制锁存器使能端
    • P2.5 控制段选锁存器（LE = Y6C （110），由Y6选通）
    • P2.7 控制位选锁存器（LE = Y7C（111），由Y7选通）
  • Y4~Y7：由74HC138译码器输出，用于选通不同锁存器

✅ 段选锁存器使能：选中 Y6 → Y6=0 → Y6C=1 → 段选锁存器打开
✅ 位选锁存器使能：选中 Y7 → Y7=0 → Y7C=1 → 位选锁存器打开

8051单片机定时器相关寄存器总结

缩写 全称 说明 相关代码示例及解释
AUXR Auxiliary Register (辅助寄存器) 某些增强型8051单片机（如STC系列）特有的寄存器，用于控制定时器的工作模式。 AUXR &= 0x7F; 将第7位（通常为T0x12位）清零，选择12T模式（即12个时钟周期为一个机器周期，传统8051模式）。
TMOD Timer Mode Register (定时器模式寄存器) 用于设置定时器0和定时器1的工作模式（如方式0~3）。 TMOD &= 0xF0; 表示清除低4位，即定时器0的模式被设为方式0（13位定时器）或方式1（16位定时器）。具体取决于后续是否设置M1/M0位。
TL0 Timer 0 Low-byte (定时器0低8位) 定时器0的低字节计数寄存器。与TH0共同组成一个16位的计数器。 (无直接代码示例，通常与TH0一起赋值)
TH0 Timer 0 High-byte (定时器0高8位) 定时器0的高字节计数寄存器。与TL0共同组成一个16位的计数器。 (无直接代码示例，通常与TL0一起赋值)
TF0 Timer 0 Overflow Flag (定时器0溢出标志位) 当定时器0计数溢出时，硬件自动置1；需软件清零。 TF0 = 0; 手动清除溢出标志。虽然上电后通常为0，但显式清零是良好习惯。
TR0 Timer 0 Run control bit (定时器0运行控制位) 位于TCON寄存器中，用于控制定时器0的启停。 TR0 = 1; 启动定时器0开始计数。 TR0 = 0; 停止定时器0计数。
ET0 Enable Timer 0 Interrupt (使能定时器0中断) 位于IE（Interrupt Enable）寄存器中，用于允许或禁止定时器0中断。 ET0 = 1; 允许定时器0溢出时产生中断。
EA Enable All interrupts (全局中断使能位) 位于IE（Interrupt Enable）寄存器中，用于全局开启或关闭所有中断。 EA = 1; 开启CPU总中断允许，否则所有中断都被屏蔽。
